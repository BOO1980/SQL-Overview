SQL Comments

  • Comments let's you include text that you don't want to be processed or executed
  • SQL statements can be short and simple but can also be quite lengthy and complex. Including descriptive comments helps for future reference
  • Comments can also contain programmer contact information and a description and notes
  • Another important use for comments is to temporarily stop SQL code from being executed
  • Comments may be embedded in-line using -- (two hyphens)

Examples

Hyphenated comment

select prod_name --this is a comment

from Products;

SQL statement

Another type of comment

select prod_name /*this is a comment*/

from Products;

SQL statement
